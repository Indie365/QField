QFieldSync plugin¶

Plugin QFieldSync pomáhá při přípravě a balení projektů QGIS pro QField.

QFieldSync supports your project preparation with automating the following:

  • Required steps for project setup (e.g. Movable project)
  • Vytváření základů z jedné rastrové vrstvy nebo ze stylu definovaného v motivu mapy.
  • Konfigurace funkce úprav offline a zpětná synchronizace změn.

Workflow¶

Chcete-li získat rychlý přehled o procesu, uvádíme seznam typických kroků:

  1. Vytvořte balíček QField. Toto je pracovní kopie v samostatné složce.
  2. Copy the QField package to the target device.
  3. Jděte ven a sbírejte data.
  4. Zkopírovat upravená data zpět do stolního počítače.
  5. Synchronizujte upravená data s databází nebo se soubory.

Instalace¶

In QGIS, open the plugin library and search for qfield sync. Select the plugin in the list and click on Install.

Konfigurace¶

Konfigurace projektu je uložena v hlavním souboru projektu .qgs. Tímto způsobem je možné projekt předběžně nakonfigurovat a použít opakovaně.

Konfigurace vrstvy¶

In the project configuration dialog, an action can be defined for each layer individually. Depending on the layer type, different types of actions are available.

Kopírovat
Vrstva bude zkopírována do složky balíčku. To je k dispozici pouze pro vrstvy založené na souborech.
No action
Zdroj vrstvy zůstane nedotčen. To je k dispozici pouze pro vrstvy bez souborů, jako jsou WMS, WFS, Postgis …
Offline editing
Pracovní kopie vrstvy se zkopíruje do složky balíčku. Každá změna, která se provádí v zabaleném projektu během práce, se zaznamenává do changelogu. Při pozdější synchronizaci změn se tento protokol přehraje a všechny změny se použijí také v hlavní databázi. Neexistuje žádné řešení konfliktů na místě.
Odstranit
Vrstva bude odstraněna z pracovní kopie. To je užitečné, pokud je vrstva použita v základním souboru a nebude k dispozici v zabaleném projektu.

Base map configuration¶

Základní mapa je rastrová vrstva, která je přidána jako nejspodnější vrstva do zabaleného souboru projektu.

Pokud je volba základní mapy povolena, bude základní mapa vykreslena vždy, když je projekt zabalen. Oblast zájmu - rozsah, který bude poskytnut - bude vybrána v době balení.

Pro základní mapu existují dva možné zdroje:

Vrstva
Rastrová vrstva. To je užitečné pro offline kopii online vrstvy, jako je WMS, nebo pro pracovní kopii nepodporovaného formátu, jako je ECW nebo MrSID vrstva.
Map Theme
Téma mapy. To je užitečné pro vytvoření základní mapy založené na kombinaci několika vrstev se stylem. Tyto vrstvy mohou být poté vyjmuty z pracovního balíčku a není nutné je na zařízení vykreslovat. To může na zařízení ušetřit místo na disku a baterii.

Velikost dlaždice definuje prostorové rozlišení. Určuje počet mapových jednotek na pixel. Pokud má mapové plátno CRS metry jako jednotky a velikost dlaždic je nastavena na 1, každý rastrový pixel bude mít prostorový rozsah 1x1 m, pokud je nastaven na 1000, bude mít každý rastrový pixel prostorový rozsah 1 čtvereční kilometr.

Konfigurace offline úprav¶

Pokud je zaškrtnuto pouze synchronizovat prvky v oblasti zájmu, budou do offline pracovní kopie zkopírovány pouze prvky, které jsou v rámci času tvorby balíčku výřezu mapy.

Package for Qfield¶

Jakmile je projekt nakonfigurován, zabalte jej do složky. Tato složka bude obsahovat soubory .qgs a data v ní použitá.

Zkopírujte složku v zařízení. Otevřete Qfield, otevřete projekt a začněte sbírat data.

Make also sure to save the QGIS project using the regular Save As of QGIS as you’ll have to re-open it later when you want to synchronize the changes.

Synchronizovat z Qfield¶

Pokud chcete synchronizovat to, co jste shromáždili, znovu otevřete projekt v QGIS (ten, který jste uložili s běžným Uložit jako).

Copy the project folder from your device to your computer and use the Synchronize from Qfield menu to synchronize your changes from the protable project to the main project.

Make sure that you synchronize your data back only once. That means, if you go out again to collect more data, you should create a new QField package before to avoid sync problems later on (like e.g. duplicates).
