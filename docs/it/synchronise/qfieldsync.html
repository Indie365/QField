Plugin QFieldSync¶

Il plugin QFieldSync aiuta a preparare ed «impacchettare» il progetto di QGIS all’uso in QField.

QFieldSync facilita la preparazione del tuo progetto attraverso l’automatizzazione dei seguenti punti:

  • Passaggi richiesti per il setup iniziale del progetto (es. Progetto «portabile»)
  • Creazione di una mappa di base a partire da un singolo raster o da uno stile definito da un «Tema Mappa» salvato
  • Configurazione della funzionalità di editing offline e sincronizzazione dei cambiamenti.

Procedura¶

Per avere una visione d’insieme del processo, riportiamo una lista dei passaggi principali:

  1. Creazione di un pacchetto per QField. Questo passaggio crea una copia del progetto, in una cartella separata.
  2. Copiare il pacchetto QField sul proprio dispositivo.
  3. Andare sul campo e raccogliere i dati.
  4. Copiare i dati raccolti sul campo all’interno del proprio computer.
  5. Sincronizzazione dei dati raccolti con il proprio database o file.

Installazione¶

In QGIS, aprire la libreria dei plugin e cercare qfield sync. Selezionare il plugin dalla lista e cliccare su Installa plugin.

Configurazione¶

La configurazione del progetto è salvata nel file master .qgs. In questo modo è possibile pre-configurare il progetto una volta sola e riutilizzarlo infinite volte.

Configurazione dei layer¶

Nella finestra di configurazione del progetto, una azione può essere definita per ogni singolo layer del progetto. A seconda del tipo di layer, è possibile definire diverse azioni.

Copia
Il layer sarà copiato nella cartella del progetto. Questa azione è possibile solamente per i layer con elementi «fisici» come shapefile o geopackage.
Nessuna azione
Il layer sorgente non sarà modificato. Questa opzione è possibile solo per quei file «non fisici» tipo WMS, WFS, Postgis…
Editing offline
Una copia del layer viene creata nella cartella del progetto. Ogni modifica che viene apportata durante il lavoro viene registrata in un file log. Durante la sincronizzazione con il progetto originale, viene letto il file log e tutti i cambiamenti vengono applicati al database di partenza. In questo modo non si creano conflitti.
Rimuovi
Il layer viene rimosso dal progetto. Questa azione è utile se un layer è usato per visualizzare informazioni ma non è utile in fase di rilievo sul campo.

Configurazione della mappa di base¶

Una mappa di base è un file raster che viene aggiunto come layer di sfondo al progetto.

Se l’opzione «Mappa di base» è abilitata, verrà renderizzata una mappa di base ogni volta che il progetto viene impacchettato. L’area di interesse - la parte visibile del progetto - verrà scelta e caricata in base alla necessità

Ci sono due possibili opzioni per la mappa di base:

Layer
Un layer raster. Opzione utile per ottenere una copia offline di una mappa online come i WMS, o per creare una copia di un tipo di file non supportato tipo WMS o MrSID.
Tema Mappa
Questa opzione è utile per creare una mappa di base definita dalla combinazione di più layer con una propria vestizione. Questi ultimi possono essere rimossi dal pacchetto senza il bisogno di essere renderizzati all’interno del progetto. Questa operazione permette di risparmiare spazio di memoria e batteria del proprio device.

La voce «Tile size» definisce la risoluzione spaziale. Determina infatti il numero di unità di mappa per pixel. Se l’unità di mappa del CRS è espressa in metri e la «tile size» è 1, ogni pixel corrisponderà ad 1x1 metri, se settato su 1000, ad un pixel corrisponderà un’estensione di un chilometro quadrato.

Configurazione dell’editing offline¶

Se è attiva l’opzione «sincronizza solo gli elementi dell’area di interesse», solo gli elementi che ricadono all’interno dell’estensione della mappa saranno disponibili per l’editing offline.

Pacchetto per QField¶

Una volta che il progetto è stato configurato, viene trasferito all’interno di un’unica cartella. Quest’ultima conterrà il progetto .qgs e tutti i file necessari al suo corretto funzionamento.

Copiate la cartella sul vostro device. Avviate QField, richiamate il progetto e iniziate l’attività di censimento.

Assicurati di salvare il progetto di QGIS utilizzando il comando «Salva come» così da poterlo richiamare successivamente per sincronizzare i cambiamenti.

Sincronizzazione da QField¶

Quando vuoi sincronizzare quello che hai censito, apri nuovamente il progetto con QGIS (quello salvato con il comando «Salva come»).

Copia la cartella del progetto dal tuo device al tuo computer e utilizza il bottone «Sincronizza da QField» per sincronizzare i cambiamenti dal progetto «portatile» a quello principale.

Make sure that you synchronize your data back only once. That means, if you go out again to collect more data, you should create a new QField package before to avoid sync problems later on (like e.g. duplicates).
